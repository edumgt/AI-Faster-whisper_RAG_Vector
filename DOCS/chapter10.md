# Chapter 10. 프론트엔드(Tailwind)와 실제 운영 확장 가이드

## 1) 프론트 구조
`api/frontend/index.html`은 Tailwind CDN 기반 단일 페이지입니다.
주요 섹션:
1. 텍스트 ingest
2. 오디오 ingest
3. 리포트 조회

모든 액션은 `fetch`로 백엔드 API를 직접 호출합니다.

---

## 2) 프론트 사용 시나리오
1. 텍스트 ingest에서 `client_id`, 상담 문장 입력 후 저장
2. 반환된 `session_id` 복사
3. 리포트 조회에 `client_id/session_id` 입력
4. 하단 `pre` 영역에서 JSON 결과 확인

---

## 3) 운영 확장 체크리스트

### 보안
- 인증/인가(JWT, RBAC)
- CORS 제한(현재는 `*` 허용)
- 업로드 파일 형식/용량 검증 강화

### 개인정보
- transcript 암호화 저장
- 민감정보 마스킹
- 접근 감사 로그

### 품질
- 분석 근거 추적성(왜 risk_score가 나왔는지)
- LLM 프롬프트 버전 관리
- A/B 테스트(모델/프롬프트 개선)

### 인프라
- API/VectorDB 모니터링
- 배치 재임베딩 파이프라인
- 장애 시 fallback/재시도 정책

---

## 4) 예시: 제품화 로드맵
- **1단계**: 현재 데모 안정화 + 테스트 자동화
- **2단계**: 사용자 인증/감사로그/비밀관리
- **3단계**: 도메인 데이터셋 기반 분석기 튜닝
- **4단계**: 상담사 워크플로우에 맞춘 UI/리포트 템플릿 고도화

---

## 5) 마무리
이 저장소는 "AI 상담 보조 시스템"의 핵심 블록(수집→분석→검색→리포트)을 짧은 코드로 체험하기에 매우 좋은 출발점입니다.
실서비스 전환 시에는 기술적 완성도뿐 아니라 윤리/보안/임상 프로토콜을 반드시 함께 설계해야 합니다.
